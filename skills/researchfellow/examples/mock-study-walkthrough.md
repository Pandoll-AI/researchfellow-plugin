# End-to-end rehearsal — mock retrospective cohort

A full 1→11 dry run of the harness on **synthetic data and a mock hypothesis**
(no PHI). It exercises the state machine, the hard gates, the honest analysis
subsystem, and the reporting coverage screen. Reproduces the P0.5 rehearsal.

> Everything here is MOCK. The cohort is generated by a seeded script; numbers are
> illustrative and not a clinical finding.

## Mock hypothesis (PICO)
- **P** adults hospitalized with community-acquired pneumonia (CAP)
- **E** early macrolide + beta-lactam **combination** therapy
- **C** beta-lactam **monotherapy**
- **O** 30-day all-cause mortality
- Design: retrospective EMR cohort → **STROBE + RECORD**; primary **Cox PH**, confounding **IPTW**.

## Mock cohort
Seeded synthetic EMR extract, n=1200 (combination 477, monotherapy 723), 160 deaths.
Columns: `exposed, event, time, age, sex, curb65, charlson, icu`. Severity (CURB-65, ICU)
drives both treatment choice (confounding by indication) and outcome; true HR ≈ 0.75.

## Transcript

### 1. PHI screen (local, offline) → CLEAN
```
$ phi_screener.py --data-path data.csv --output .research/phi-report.json
  Findings: 0  Max severity: clean
```

### 2. Planning (steps 1–8) + state judgments
Artifacts written: `idea.json, literature.json, evidence-table.json, variables.json,
protocol.md, sap.md, shells.md`. The read-only judge confirms structure and gates:
```
$ state_tool.py validate --project-dir .research/          → {"schema":"v2","violations":[]}
$ state_tool.py can-enter --step 5                          → allowed=True  (soft gate.endpoint warned)
$ state_tool.py can-enter --step 9                          → allowed=False missing_hard_gates=[gate.feasibility, gate.protocol]
```

### 3. Synthetic dry-run (planning) — honest, no false precision
```
$ analysis_runner.py --mode synthetic --project-dir .research/
  glm status: aggregate_only   *** NOT REAL DATA ***
```
Aggregate input returns `aggregate_only` — **no fabricated CI/p**.

### 4. Emit the reproducible analysis script (plan mode)
```
$ analysis_runner.py --mode plan --plan-path .research/analysis-plan.json --data-path data.csv
  Script (authoritative, run in R): .research/analysis/scripts/analysis.R
  [INFO] ph_assumption: verify proportional hazards (cox.zph)
```
The emitted `analysis.R` sets up IPTW (`weightit(exposed ~ age + sex + curb65 + charlson + icu ...)`,
balance via `bal.tab`), a `coxph(Surv(time, event) ~ ...)`, `cox.zph`, and an E-value stub.
The tool emits code; it does not fabricate numbers.

### 5. Hard gate is fail-closed (the brand promise, live)
Attempting real analysis **before** the gates are approved:
```
$ analysis_runner.py --mode real --project-dir .research/ --data-path data.csv
  ERROR: Missing required real-data gate approvals: ['gate.feasibility','gate.protocol','gate.qc']
  → exit 1   (blocked)
```
QC (`qc_checker.py`) passes with `has_critical=false`; the three hard gates are then
approved by the user and recorded in `state.json`:
```
$ state_tool.py gate-check --for real-analysis   → ok=True
$ state_tool.py can-enter --step 10              → allowed=True
```

### 6. Real analysis (individual-level, genuine inference)
With the gates approved and the stats stack installed:
```
$ analysis_runner.py --mode real --project-dir .research/ --data-path data.csv
  cox     : ok  HR=0.704  95% CI [0.507, 0.978]  p=0.0361
  logistic: ok  OR=0.792  95% CI [0.559, 1.121]  p=0.1879  (crude preview)
  table1  : exposed 477 (56 events) / unexposed 723 (104 events)
```
The Cox model recovers the mock HR (~0.75) with a **real** CI/p. The Python logistic is a
crude preview (unadjusted); the **authoritative** IPTW-adjusted estimate comes from the
emitted R script.

### 7. Reporting coverage (design-aware, free)
```
$ checklist_map.py --design cohort --manuscript .research/manuscript.md
  Checklist coverage (STROBE, RECORD, design=cohort):   ← RECORD auto-pulled (EMR/ICD-10)
  covered=26 unclear=9 missing=1 / 36
  REQUIRED items needing attention: STROBE-3, STROBE-8, STROBE-18
```
The screen correctly pulls in RECORD from the EMR/ICD-10 wording and flags real gaps in
the draft (objectives/hypothesis, data-source detail, key-results summary).

### 8. Final state
```
$ state_tool.py validate --project-dir .research/   → schema=v2, violations=0
```

## What this rehearsal demonstrates
- PHI stays local and screens clean before anything else.
- The state machine judges structure deterministically; steps needing hard gates are
  blocked until those gates are approved (`exit 1`), from any entry path.
- Analysis is honest: aggregate → point estimate only; individual data → real CI/p; the
  reproducible R script is the authoritative artifact.
- Reporting coverage is design-aware and actionable.
